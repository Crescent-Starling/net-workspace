# 开发说明

## 1. 开发语言

默认使用中文进行：

- 产品需求
- 架构说明
- 设计讨论
- Issue 和 PR 描述
- 提交说明

以下内容保留英文：

- 代码标识符
- API 路径
- 数据库表名和字段名
- 目录名
- 必要的第三方技术术语

## 2. 开发顺序建议

### 第一阶段：骨架搭建

- 完善文档
- 初始化前端工程
- 初始化后端工程
- 建立数据库迁移体系

### 第二阶段：核心链路

- 用户与工作区
- 模板体系
- 采集任务体系
- 数据落库

### 第三阶段：智能能力

- 智能站点接入
- 模板测试
- AI 字段抽取
- 工作区 Agent

### 第四阶段：分析与演化

- 仪表板
- 趋势分析
- 提案与审核
- 共享能力库

## 3. 文档维护要求

以下变更必须同步更新文档：

- 数据库结构变化
- API 变化
- 模板 DSL 变化
- Agent 行为变化
- 工作流变化

## 4. 目录职责

### frontend/

前端工作台实现。

### backend/

后端 API、业务逻辑、任务编排、Agent 接口。

### spiders/

采集器、模板执行器、站点适配逻辑。

### docs/

产品、架构、数据库、协作与开发说明。

### tests/

单元测试、集成测试、端到端测试。
